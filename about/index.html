※ NavBar 기능과 사용

   

1. Home  

     - 블로그의 메인화면으로 이동합니다.

     - 사용자가 지정한 블로그의 Feature Post가 출력됩니다.


2. About  

     - 블로그의 도움말 페이지로 이동합니다.


3. Category Post  

     - 카테고리 별 포스트를 출력하는 화면으로 이동합니다.

     - 카테고리 별로 포스트이 작성날짜에 따라 배열됩니다.


4. All post  

     - 모든 포스트를 출력합니다.

     - 작성날짜에 따라 포스트들이 배열됩니다.


5. Tag 별 post  

     - 태그(주제)별 도움말 페이지로 이동합니다.

     - 주제별로 작성된 포스트들의 대표내용이 모두 출력됩니다.

※ 편집자 페이지

 -> 이름, 닉네임, 지위, 자기소개, 이메일, 소속, ID, 가입일, 소셜

- Hydn의 연속기획물인 시리즈 Post를 출력하는 페이지입니다.       TIP) 시리즈 포스트의 회색글씨는 링크입니다.

※ Google Search Console 활용하기

- Google Search Console은 검색엔진 Google에 본인의 사이트를 등록해 더 많은 방문자를 유입되게 하는 기능입니다.

해당 과정으로, 보유한 Page를 Google 검색엔진에 노출시켜 작성한 글이 다른 사람에게 쉽게 노출되도록 처리할 수 있습니다. 이 작업을 하지 않으면 작성된 글이 웹에 노출될 확률이 현저하게 떨어집니다.


Google이 제공하는 Google Search Console을 이용하여 내 GitHub Page의 Domain을 등록시키고 sitemap.xml을 생성해 Google Search Console에 제출하면 됩니다. sitemap.xml을 등록해 두면 Google 검색 크롤러가 주기적으로 페이지를 크롤링하여 indexing하게 됩니다.


1. 먼저 현재 블로그 root밑에 robots.txt가 존재할 수 있도록 파일을 생성합니다. (plugin : jekyll-sitemap 사용 시 자동생성됩니다.)

   - 관련 링크는 오른쪽을 참조해주세요. >>> robots.txt 알아보기


2. 그런다음 sitemap.xml을 작성해야 합니다. sitemap.xml 역시 root밑에 존재해야 하며 추후에 Google Search Console에 제출해야 합니다. (마찬가지로 plugin : jekyll-sitemap 사용 시 자동생성됩니다.)

3. sitemap.xml 작성이 끝났으면 이제 Google Search Console 에 접속합니다.
